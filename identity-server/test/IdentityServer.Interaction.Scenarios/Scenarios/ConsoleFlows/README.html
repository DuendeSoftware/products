ConsoleClientCredentials — OAuth 2.0 Client Credentials Flow

Demonstrates a machine-to-machine client obtaining an access token from Duende IdentityServer using the OAuth 2.0 Client Credentials grant, then calling a protected API with that token. No user login or browser redirect is involved.

Services

Two in-process hosts start when this scenario runs:

Service Name Role
Duende IdentityServer identity-server Issues access tokens
Protected API api Resource server — requires a valid bearer token

Client configuration

Setting Value
client_id client
client_secret secret
Grant type client_credentials
Allowed scopes resource1.scope1, resource1.scope2

No user context is associated with the token — the client authenticates as itself.

Flow

  1. Fetch the OpenID Connect discovery document from IdentityServer to locate the token endpoint.
  2. POST to the token endpoint with client_id, client_secret, and grant_type=client_credentials.
  3. IdentityServer returns an access token (JWT).
  4. Send a GET /identity request to the API with the token in an Authorization: Bearer header.
  5. The API returns the claims from the access token as a JSON array.

API endpoint

Endpoint Method Description
/identity GET Returns the claims present in the bearer token

Key implementation details

  • No interactive login — the client authenticates directly with its credentials; there is no authorization code exchange or redirect
  • Token audience — the access token is scoped to resource1.scope1 and/or resource1.scope2; request only the scopes needed
  • TLS — the test hosts use self-signed certificates; certificate validation is bypassed in the scenario client
